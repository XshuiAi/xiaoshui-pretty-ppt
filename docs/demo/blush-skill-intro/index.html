开场先讲痛点：不是再做一个普通网页模板，而是把文档变成可以直接演示、改动和分享的网页 PPT。
Why this exists

普通 AI PPT 最大的问题，是好看但不一定可用。

这套 skill 解决的是真实交付问题：中文清晰、结构可控、生成后还能继续修改和发布。

图片式生成

看起来像 PPT，但本质是图片。中文放大容易糊，文字和版式很难二次调整。

模板靠抽卡

每次都要赌结果，适合展示的很少，适合正式汇报的更少。

交付不连续

文档、PPT、网页、发布链接彼此割裂，内容改一遍，视觉还要再排一遍。

这里不要贬低所有 AI PPT，而是说这套技能专门补齐可编辑、可复用、可发布三个环节。
Workflow

从飞书文档到网页 PPT，中间只保留必要判断。

先判断受众、场景和内容密度，再选模板、拆页面、生成 HTML，最后按 E 修改或按 P 演讲。

飞书文档 / 文章 / 笔记

把原始材料交给 Agent，不要求用户先手动改成 PPT 大纲。

场景与密度判断

区分演讲型、分享型、报告型、教程型，不把长文硬塞进一页。

套用真实模板

从 12 套模板复制开始，保留配色、版式、导航和动效。

打开、编辑、发布

本地浏览器打开；需要时按 E 改文字，按 P 进入演讲者模式。

这里可以把它讲成一条流水线：不是一次性图片，而是一个能继续迭代的 presentation workflow。

PromptDocument to deck
使用 $xiaoshui-pretty-ppt，把这个飞书文档做成一份适合对外分享的 HTML 网页 PPT。
要求：
1. 先判断内容更适合哪一套模板；
2. 把长文档拆成封面、钩子、核心观点、案例、流程、总结；
3. 开启可编辑模式，方便我录视频时展示生成后还能改；
4. 最后告诉我本地 HTML 路径和如何打开。
这里可以直接复制给观众。重点是让用户不用懂模板细节，只需要讲明内容、场景、密度和是否要编辑。
Presenter mode

下一步不是只做页面，而是补齐演示能力。

这版已经加入基础演讲者模式：按 P 打开讲稿备注、下一页预览和计时器；按 E 仍然可以进入编辑模式。

P

演讲者模式

展示当前页标题、摘要、下一页标题、讲稿备注和演讲计时。

E

浏览器编辑

生成后可直接点文字修改，再导出一份改好的 HTML 文件。

HTML

发布链接

静态文件可以放到 GitHub Pages、Cloudflare Workers 或任意静态托管。

现场可以按 P 演示这个页面本身的演讲者模式，让大家看到它不是静态截图。
Roadmap

后续优化会围绕四个更高级的 presentation 能力展开。

不急着把 PPT、HTML、飞书画板混成一个技能。先把 HTML 网页 PPT 做深，再拆出其他形态。

01 Nested slides

横向主线 + 纵向详情

适合课程、产品介绍和长报告，把补充材料放到二级页面。

02 Auto Animate

章节之间自动过渡

让同一张图、同一个卡片在前后页之间自然移动，减少硬切。

03 Data report

数据汇报型 PPT

把 KPI、趋势、漏斗、对比和结论做成可读的报告页。

04 Cinematic portfolio

电影播放感作品集

用全屏镜头、章节字幕和滚动叙事，把 HTML PPT 做成高级作品集网站。

这里回应用户关心的前沿实践：Reveal.js 的标准能力可以分阶段吸收，但先做最容易转化的演讲者模式和数据页。
Final message

这不是把内容装进模板，而是把内容变成可以被讲出来的作品。

xiaoshui-Pretty PPT 的重点，是让文档、演示、编辑和发布变成同一条链路。
